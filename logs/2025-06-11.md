## 오늘 품었던 의문점 & 배운 개념

### 🤔 Hooks · Rules · Agent · MCP 차이가 뭐야?

| 개념 | 역할 | 이 프로젝트에서 쓰임 |
|------|------|-------------------|
| **CLAUDE.md** | 프로젝트 컨텍스트 자동 로드 | 매 대화마다 구조·규칙 인식 |
| **Rules** | 특정 작업의 행동 패턴 지정 | 리뷰 형식, 이론 템플릿, 진도 포맷 |
| **Hooks** | 이벤트 발생 시 자동 실행 스크립트 | 저장시 컴파일, 하루 마무리 요약 |
| **Agent** | 여러 작업을 자율적으로 연속 실행 | "Day 완료 처리하고 README 갱신해줘" |
| **MCP** | 외부 서비스 연결 (GitHub, Notion 등) | GitHub 연동으로 커밋·PR 자동화 |

### 🤔 Claude Code 어떻게 활용해?

| 용도 | 방법 |
|------|------|
| 이론 정리 | "BFS 이론 md 만들어줘" → `theory/` 저장 |
| 풀이 리뷰 | 코드 붙여넣기 → 시간복잡도·개선점 피드백 |
| 진도 업데이트 | "Day 5 완료 처리해줘" → PROGRESS·README 자동 갱신 |
| 오답 분석 | 틀린 코드 → 왜 틀렸는지 + 수정 코드 |
| Agent 활용 | 멀티스텝 자동화 ("오늘 정리 다 해줘") |

### 🤔 세팅 우선순위는?

```
1순위  CLAUDE.md + Rules    ← 오늘 완료 ✅
2순위  Git + GitHub 연동    ← 다음 스텝
3순위  Hooks 활성화         ← Claude Code 설치 후
4순위  MCP 연동             ← GitHub MCP
```
