# Rule: 풀이 리뷰

풀이 코드 리뷰 요청 시 아래 형식으로 응답한다.

## 리뷰 출력 형식

```
╔══════════════════════════════════╗
║  CODE REVIEW                     ║
║  문제: {문제명}                    ║
║  날짜: {날짜}                      ║
╚══════════════════════════════════╝

[시간복잡도] O(?)
[공간복잡도] O(?)

[잘한 점]
- ...

[개선 포인트]
- ...

[개선 코드]
\`\`\`java
// 개선된 코드
\`\`\`

[추천 문제]
- 프로그래머스 Level? - {문제명}
```

## 규칙

- 시간복잡도는 반드시 명시 (모르면 추정치라도)
- 개선 코드는 실제 실행 가능한 Java 코드로
- 칭찬보다 개선점에 집중
- 추천 문제는 같은 유형으로 1~2개
