# Rule: 진도 업데이트

진도 체크 요청 시 아래 순서로 처리한다.

## 처리 순서

1. `PROGRESS.md` 에서 해당 Day 찾아서 `[ ]` → `[x]` 체크
2. `README.md` 진도율 테이블 갱신
3. 완료된 Day 수 계산해서 전체 진도율 업데이트
4. Git 커밋 메시지 제안 출력

## 진도율 계산

```
전체 진도율 = 완료된 Day / 40 * 100
섹션별 진도율 = 완료된 섹션 Day / 섹션 총 Day * 100
```

## 커밋 메시지 포맷

```
[DayN] 완료 - {오늘 주제}

- 학습: {이론 내용}
- 풀이: {문제명} x{개수}
- 메모: {특이사항}
```

## README 진도 테이블 형식

```markdown
| 섹션 | 내용 | 진도 | 상태 |
|------|------|------|------|
| Phase 1 | Java 기초 | 3/5 | 🟡 진행중 |
| Phase 2 | 기본 알고리즘 | 0/10 | ⬜ 대기 |
```

상태 이모지:
- ✅ 완료
- 🟡 진행중  
- ⬜ 대기
- 🔴 지연
